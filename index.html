Java Básico: Saída e Variáveis

📤 Saída de dados

Em Java, usamos System.out.println() para imprimir uma linha (com quebra) e System.out.print() para imprimir sem quebrar a linha.

exemplo das aulas
public class OlaMundo {
    public static void main(String[] args) {
        System.out.println("Olá, Mundo!");   // imprime e quebra linha
        System.out.print("Sem quebra");       // não quebra linha
    }
}

Entrada de Dados — Scanner

⌨️ Usando o Scanner

O Scanner é a classe usada para ler dados do teclado. Vimos isso em várias aulas: Saudacoes2, NomesConjuge, retângulo, círculo etc.

import java.util.Scanner; // SEMPRE importar!
 
public class Exemplo {
    public static void main(String[] args) {
        Scanner teclado = new Scanner(System.in);
 
        System.out.print("Qual seu nome? ");
        String nome = teclado.nextLine();  // lê texto
 
        System.out.print("Qual sua idade? ");
        int idade = teclado.nextInt();     // lê inteiro
 
        System.out.print("Qual seu salário? ");
        double sal = teclado.nextDouble(); // lê decimal
 
        System.out.println("Olá, " + nome + "!");
    }
}
⚠️ Atenção: Use nextLine() para String, nextInt() para inteiros, nextDouble() para decimais. No Brasil o separador decimal pode ser vírgula!
📝 Padrão visto nas aulas: Nomes + Salários
// Padrão visto na aula com leNome() e leSalario()

double salario1 = leSalario(nome1);
 
System.out.println(nome1 + " ganha R$ " + salario1);

Métodos auxiliares estáticos (static) foram criados para ler nome e salário, deixando o código mais organizado — isso é modularização!

Classes e Objetos

💡 Conceito Central de POO

Uma classe é o molde/projeto. Um objeto é a instância criada a partir desse molde. Assim como uma planta de uma casa (classe) pode gerar várias casas construídas (objetos).

🎓 Classe Aluna — vista em aula (UML + Código)

No quadro vocês viram o diagrama UML e depois o código. Veja a correspondência:

diagrama UML → código Java
// Atributos (campos da classe)
public class Aluna {
    private String ra;
    private String nome;
    private int    idade;
    private String crm;
    private double mensalidade;
 
    // Construtor
    public Aluna(String ra, String nome, int idade) {
        this.ra    = ra;
        this.nome  = nome;
        this.idade = idade;
    }
 
    // Getter
    public String getNome() { return nome; }
 
    // Setter
    public void setNome(String nome) { this.nome = nome; }
}
 
// Criando objetos
Aluna a1 = new Aluna("001", "Maria", 20);
Aluna a2 = a1; // a2 aponta para o MESMO objeto que a1!
⚠️ Cuidado com referências: No código da aula, Aluna a2 = a1 não cria uma cópia — ambas apontam para o mesmo objeto na memória!
06

Construtores, Getters e Setters

🏗️ Encapsulamento com private + getters/setters

Atributos são declarados como private (encapsulamento). O acesso externo é feito via métodos públicos: get (retorna valor) e set (altera valor).

public class Pessoa {
    private String nome;  // privado — não acessível diretamente
    private int idade;
 
    // Construtor
    public Pessoa(String nome, int idade) {
        this.nome  = nome;
        this.idade = idade;
    }
 
    // Getter — retorna o valor
    public String getNome()  { return nome; }
    public int    getIdade() { return idade; }
 
    // Setter — altera o valor
    public void setNome(String nome)  { this.nome  = nome; }
    public void setIdade(int idade)   { this.idade = idade; }
}
🔑 A palavra-chave this

this referencia o próprio objeto atual. É usado no construtor e setters para diferenciar o atributo da classe do parâmetro do método quando têm o mesmo nome.

public void setNome(String nome) {
    this.nome = nome; // this.nome = atributo, nome = parâmetro
}
07

Herança e super()

🧬 Herança — extends

Nas aulas vocês viram a classe Aluna herdando de Pessoa. A classe filha herda atributos e métodos da classe pai.

// Classe pai (superclasse)
public class Pessoa {
    private String nome;
 
    public Pessoa(String nome) {
        this.nome = nome;
    }
    public String getNome() { return nome; }
}
 
// Classe filha (subclasse) — usa extends
public class Aluna extends Pessoa {
    private String ra;
 
    public Aluna(String ra, String nome) {
        super(nome); // chama o construtor da classe PAI!
        this.ra = ra;
    }
}
 
// Uso
Aluna a = new Aluna("001", "Maria");
System.out.println(a.getNome()); // herdado de Pessoa
super() deve ser a PRIMEIRA instrução no construtor da classe filha. Ele invoca o construtor da superclasse.
08

Os 4 Pilares da POO

🔒
Encapsulamento
Esconder atributos com private e acessar via get/set. Protege os dados internos.
🧬
Herança
Uma classe filha herda de uma classe pai com extends. Reutiliza código.
🎭
Polimorfismo
Mesmo método com comportamentos diferentes em classes filhas. Sobrescrita com @Override.
📐
Abstração
Modelar o mundo real em classes, expondo apenas o essencial. Classes abstratas e interfaces.
🎭 Polimorfismo — Sobrescrita de Método
public class Pessoa {
    public void apresentar() {
        System.out.println("Sou uma pessoa.");
    }
}
 
public class Aluna extends Pessoa {
    @Override
    public void apresentar() { // mesmo método, comportamento diferente
        System.out.println("Sou uma aluna!");
    }
}
 
// Polimorfismo em ação:
Pessoa p = new Aluna(); // referência Pessoa, objeto Aluna
p.apresentar();           // imprime "Sou uma aluna!"

🟣 Múltipla Escolha

M-01
Qual instrução lê uma linha de texto completa com o Scanner?
a) scanner.next()
b) scanner.nextLine()
c) scanner.nextString()
d) scanner.read()
Resposta: b) nextLine() — a única que lê a linha inteira, incluindo espaços.
M-02
Dado: Aluna a2 = a1;. O que acontece se você alterar a2.setNome("Joana")?
a) Só o nome de a2 muda
b) Só o nome de a1 muda
c) Tanto a1 quanto a2 têm o nome alterado para "Joana"
d) Ocorre um erro de compilação
Resposta: c) — a1 e a2 apontam para o MESMO objeto. Alterar por uma referência altera para as duas.
M-03
Qual é a saída do código abaixo?
int x = 10;
int y = 3;
System.out.println(x / y);
System.out.println(x % y);
a) 3.33 e 1
b) 3 e 1
c) 3.0 e 1.0
d) 4 e 1
Resposta: b) — divisão inteira entre dois int trunca o resultado. 10/3 = 3, 10%3 = 1.
M-04
Qual palavra-chave permite que uma classe herde de outra em Java?
a) implements
b) extends
c) inherits
d) super
Resposta: b) extends

🟡 Debug — Encontre e Corrija o Erro

D-01
O código abaixo não compila. Encontre e corrija TODOS os erros.
// CÓDIGO COM ERROS
public class Carro {
    private String modelo
    private int ano;

    public Carro(String modelo, int ano) {
        modelo = modelo;
        this.ano = ano;
    }

    public String getModelo {
        return modelo;
    }
}
Erros: 1) Falta ponto e vírgula após String modelo. 2) modelo = modelo não usa this, então o atributo nunca é atribuído. 3) getModelo está faltando os parênteses ().
D-02
Por que este código de herança não compila?
public class Animal {
    private String nome;
    public Animal(String nome) { this.nome = nome; }
}

public class Cachorro extends Animal {
    private String raca;
    public Cachorro(String raca) { // erro aqui!
        this.raca = raca;
    }
}
Resposta: A classe pai Animal não tem construtor padrão (sem parâmetros). O construtor de Cachorro precisa chamar super(nome) explicitamente como primeira instrução.
